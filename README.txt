08-Security - Loja de Games

Construa a Backend para uma Loja de Games com a capacidade de manipular os dados dos Produtos da loja. Os produtos deverão estar classificados por Categoria.



1.Insira todas as Dependências necessárias para a Spring Security
2.Na Camada Model, crie as duas Classes Model, necessárias para a Spring Security e coloque todos os atributos necessários, 3.incluindo os Métodos Get e Set e as Anotações. Não é necessário criar um Método Construtor.
4.Na Camada Model, crie os Relacionamentos necessários para a Spring Security. Não esqueça de acrescentar todas as Anotações e 5.Métodos Get e Set necessários nas 2 Classes.
6.Execute a sua aplicação no STS e verifique se a tabela foi criada no MySQL Workbench.
7.Na Camada Repository, crie a terceira Interface Repository.
8.Na nova Interface Repository, crie o Método de Busca Específica necessário para a Spring Security
9.Crie a Camada Security com as Classes necessárias para a Spring Security. 
10.Crie a Camada Service com as Classe de Serviço necessária para a Spring Security.  
Na Camada Controller, crie a segunda Classe Controller.
Na Classe Controller, crie todes os Métodos do CRUD.
11.Teste todes os métodos do CRUD através do Insomnia.
12.Ao concluir a atividade, envie as atualizações do projeto para o repositório remoto.
13.Na entrega das atividades na Plataforma, efetue a validação das 3 Classes e da Interface desenvolvidas no Recurso Usuario, utilizando as seguintes camadas:
Usuario: Camada model
UsuarioRepository: Camada repository
UsuarioController: Camada controller
UsuarioService: Camada service
